Welcome to the Wcfg project page.

Wcfg is mainly a shell script that can configure (or remove) complete LAMP-like virtual servers. It can manage many different things, like nginx or apache2 NameVirtualServers, mysql databases and privileges, (ftp) users, awstats static reports. logrotate will also be configured to take care of your virtual servers, but everything is keep consistent with the regulations and automatisms provided by the standard Debian packages.

An FTP server is not required, however a new user will be created for every virtual server, with the website root path as home directory.

Removing a virtual server will clean everything, but a complete backup is dumped in the current directory before leaving.

Website backups

Wcfg also provides advanced website backup feature using rsync real-link snapshots, so you won't waste disk space except for the only filesthat were modified since last backup, plus an updated database dump (if a database is used by that specific website, of course).

If a forth directory called backup exists under the website main tree, the wcfg-backup script (it usually runs from a daily cron job) will take snapshots as explained above. You can choose to exclude files or directories from your daily snapshots listing them in a file named exclude under the backup directory. You must specify relative paths starting from the website root directory, like:

htdocs/uploads/tmp

If you use views for any of your database, please provide the show view grant to the debian-sys-maint user with the following SQL commands after loggin into MySQL as admin user:

	use mysql;
	update user set Show_view_priv = "Y" where User = 'debian-sys-maint';
	flush privileges;
